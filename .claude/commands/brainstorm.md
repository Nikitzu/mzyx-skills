Invoke the `mzyx-skills:brainstorming` skill.

Use this **before** `/spec` whenever the request is fuzzy or under-specified. The skill will:

1. Pull out the real intent behind the request
2. Surface assumptions and constraints
3. Map alternatives and trade-offs
4. Hand back a concrete-enough problem to write a spec for

Output is exploratory — bullets, options, open questions. Do not produce code, a final plan, or a spec in this phase.

Next step in the flow: `/spec`.
